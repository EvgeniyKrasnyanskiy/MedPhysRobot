# План исправления бага с ответами админов в админ-группе

В админской группе при общении админов между собой через ответы (reply) бот пишет:
`⚠️ Не удалось найти пользователя для ответа. Возможно, сообщение слишком старое (более 14 дней).`
Это происходит потому, что бот перехватывает все реплаи в админ-группе и пытается найти в БД связь с пользователем, даже если реплай был сделан на сообщение другого админа.

То же самое происходит при редактировании любого сообщения админом в админ-группе, если для него не найдено соответствие в БД.

## Предлагаемые изменения

### 1. `handlers/relay.py`

В файле `handlers/relay.py` будут внесены изменения в два обработчика:

#### `handle_group_reply`
Добавить проверку:
```python
    # Проверяем, что ответ направлен именно на сообщение бота
    if message.reply_to_message.from_user.id != bot.id:
        return
```

#### `handle_admin_edit`
Добавить проверку:
```python
    # Проверяем, что редактируемое сообщение является ответом на сообщение бота
    if not message.reply_to_message or message.reply_to_message.from_user.id != bot.id:
        return
```

### 2. Тестирование

Добавим автотесты в `tests/test_relay.py` для имитации ситуаций:
- Ответ админа на сообщение не-бота (игнорируется).
- Редактирование админом сообщения, не являющегося ответом на сообщение бота (игнорируется).
